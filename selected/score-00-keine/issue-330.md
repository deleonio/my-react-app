## Beschreibung

Die Inbetriebnahme sollte so gut wie möglich automatisiert ablaufen. Vorherige Prüfungen wie die Durchführung automatisierter Tests, Einhaltung von Programmierrichtlinien oder Security Checks sind dabei Standard. Im folgenden eine Auswahl an Tools für die Automatisierung des Inbetriebnahme-Prozesses:

* Security Scanning z.B.
  * Renovate (https://github.com/renovatebot/renovate, GNU Affero General Public License v3.0)
  * Dependabot (https://github.com/dependabot/, MIT Lizenz)
  * Trivy (https://www.aquasec.com/products/trivy/, https://github.com/aquasecurity/trivy, Apache 2 Lizenz)
  * OWASP Dependency Check (https://owasp.org/www-project-dependency-check/, Apache 2 Lizenz)
  * Sonarqube (https://www.sonarsource.com/, LGPLv3)
* Linting z.B.
  * eslint (https://github.com/eslint/eslint, MIT License)
* Testautomatisierung z.B.
  * Selenium (https://www.selenium.dev/, Apache 2.0 license)
  * Robot Framework (https://robotframework.org/, Apache 2.0 license)
  * Cypress (https://github.com/cypress-io/cypress, MIT License)
