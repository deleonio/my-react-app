## Beschreibung

Die Definition von Standard im Glossar ist nicht zu gebrauchen: "Standards entstehen dabei aus pragmatischen Abstimmungsprozessen von Anwendern und Anbietern. Die Anwender forcieren insbesondere die Standardisierung in den Verfahrensabläufen. Die Anbieter stimmen sich zu Standards in Produkten ab."

Das klingt zu modellhaft akademisch.

De Jure Standards sind auf Deutsch die "Normen": DIN, ISO usw. 

In der IT kommen dazu Konsortialstandards (W3C, OASIS etc.), die auch in der öffentlichen Vergabe nach Verordnung 1025/2012 referenziert werden können, dort definiert als:

<table>
<tr>
<td>„technische IKT-Spezifikation“: eine technische Spezifikation im Bereich der Informations- und Kommunikationstechnologien;</td>
</tr>
</table>

Semantische Standards wie XÖV Formate oder die typischen REST API / XML / JSON / Open API Verfahren zum Datenaustausch kann man in diesen Komplex hineinziehen.

Standardisierung im Sinne von lokal betriebener Vereinheitlichung einer IT Infrastruktur, dagegen, etwa die Festlegung auf Produkte eines einzigen Herstellers, gehört natürlich nicht hinein, ist auch vergaberechtlich irrelevant.

Der europäische Interoperabilitätsrahmen sprach von "standards and specifications", auf Deutsch "Normen und Spezifikationen".

Man sollte hier schon in dem Rahmen bleiben, den das WTO TBT vorgibt, und mit Verordnung 1025/2012 in der Europäischen Union umgesetzt ist.
